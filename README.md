The main goal of this issue tracker project is to gain a deeper understanding of using a front end framework with a golang backend. Other goals are:

- Practice test driven development  (TDD)
- Utilize knowledge of formal project documentation practices tied with version control
- Gain further experience with estimations based on level of effort not time
- Gain RestAPI creation experience with golang
- Utilize a different architectural model then MVC
- Allow the application to be fully docker-ized
